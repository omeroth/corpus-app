# share/

Generated by `npm run gen:share`. The build script writes:

- `share/cards/<lang>/<weekId>-<dayId>.png` — one 1200×630 OG image per
  dialogue × language. Referenced by the `<meta property="og:image">` tag
  in the matching `d/<weekId>-<dayId>-<lang>.html` page.
- `share/cards/manifest.json` — lists every (weekId, dayId, lang, subject,
  thinkerId, title, card path, page path) tuple so the app can look up a
  card path at runtime if it ever needs to (currently unused — the app
  points at the landing page directly).

This directory (and `d/`) is copied into `_deploy/` by the Pages workflow
so `corpusapp.io` can serve the cards for OG previews. It is
**intentionally NOT included** in `www/` — the Capacitor native bundles
don't need these files (the app shares URLs at runtime, never reads the
PNGs locally), and including them was adding ~8 MB to every app update.

The generator is fully deterministic — running it again with the same
`index.html` produces byte-identical cards + pages.
